📖 이미지 최적화 가이드 05
블로그 로딩 속도와 방문자 이탈률의 관계 — 느린 블로그가 검색 순위를 망치는 이유
좋은 글을 써도 페이지가 느리게 열리면 방문자는 이미 뒤로 가기를 눌렀습니다. 로딩 속도가 블로그 SEO와 수익에 미치는 영향과 이미지로 속도를 높이는 방법을 정리합니다.
2025년 7월
숫자로 보는 로딩 속도의 영향
53%
모바일 사용자의 53%는 페이지 로딩에 3초 이상 걸리면 이탈합니다. (Google/SOASTA Research)
1초
로딩 시간이 1초 늘어날 때마다 전환율이 약 7% 감소합니다. 방문자가 광고를 클릭할 확률도 낮아집니다.
2021
구글은 2021년부터 Core Web Vitals를 검색 순위 신호로 공식 채택. 느린 페이지는 순위가 밀립니다.
블로그 속도를 가장 많이 잡아먹는 것 — 이미지
일반적인 블로그 포스팅에서 이미지가 전체 페이지 용량의 60~80%를 차지합니다. 글 본문, 광고, 스크립트를 다 합쳐도 이미지보다 작은 경우가 대부분입니다.

스마트폰 카메라로 찍은 사진 10장의 용량은 보통 40~80MB입니다. 이것을 최적화 없이 블로그에 올리면 방문자는 느린 인터넷 환경에서 10초 넘게 기다려야 할 수도 있습니다.
  • 스마트폰 사진 원본: 약 4~8MB/장
  • WebP 변환 후: 약 100~300KB/장 (90% 이상 절감)
  • 10장 기준: 50MB → 2MB 이하로 줄어듦
Core Web Vitals와 이미지의 관계
구글이 측정하는 Core Web Vitals 3가지 중 이미지와 직접 연관된 항목은 다음과 같습니다.
  • LCP (Largest Contentful Paint) — 페이지에서 가장 큰 콘텐츠(보통 대표 이미지)가 로드되는 시간. 2.5초 이하가 목표. 이미지 압축으로 가장 직접적으로 개선됩니다.
  • CLS (Cumulative Layout Shift) — 이미지가 늦게 로드되면서 글자나 버튼이 밀리는 현상. 이미지에 width, height 속성을 지정하면 예방됩니다.
LCP가 2.5초를 넘기는 블로그는 구글 PageSpeed Insights에서 낮은 점수를 받고, 동일 키워드 경쟁에서 뒤처질 수 있습니다.
💡 구글 PageSpeed Insights 확인 방법
pagespeed.web.dev에서 자신의 블로그 URL을 입력해보세요. 이미지 최적화 관련 개선 사항을 구체적으로 알려줍니다. '다음 세대 형식으로 이미지 제공'이라는 항목이 뜬다면 WebP로 전환하라는 신호입니다.
이미지 최적화 외에 속도를 높이는 방법
이미지 압축이 가장 효과적이지만, 함께 챙기면 좋은 것들도 있습니다.
  • 이미지 지연 로딩(Lazy Loading) — 화면에 보이지 않는 아래쪽 이미지는 스크롤할 때 로드. 티스토리 스킨에서 loading="lazy" 속성 추가로 적용.
  • 불필요한 플러그인·위젯 정리 — 방문자 수 카운터, 날씨 위젯 등은 외부 스크립트를 호출해 로딩을 늦춥니다.
  • 대표 이미지(썸네일) 별도 최적화 — LCP 요소가 되는 대표 이미지는 특히 작게 만드세요. 100KB 이하를 목표로 합니다.
