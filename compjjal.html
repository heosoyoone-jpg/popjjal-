아니요, 전혀 저장되지 않습니다. 압축짤은 브라우저의 Canvas API를 이용해 이미지를 처리합니다. 업로드한 파일은 외부 서버로 전송되지 않고 오직 사용자의 브라우저 메모리 안에서만 처리되며, 페이지를 닫으면 즉시 사라집니다.

네, 구글은 페이지 로딩 속도를 SEO 점수에 반영합니다. 동일한 품질의 JPG 대비 WebP는 평균 30~50% 더 작아서 페이지가 빠르게 로드됩니다. 특히 Core Web Vitals의 LCP(최대 콘텐츠 표시 시간) 개선에 직접적인 영향을 줍니다. 자세한 내용은 WebP vs PNG vs JPG 비교 가이드를 참고하세요.

글의 핵심 키워드를 한국어로 입력하세요. 예를 들어 대구 맛집 포스팅이라면 '대구맛집'을 입력하면 대구맛집-1.webp, 대구맛집-2.webp … 순서로 저장됩니다. 구글은 이미지 파일명을 크롤링하기 때문에 정확한 키워드로 작성된 파일명이 이미지 검색 노출에 도움이 됩니다. 파일명 짓는 법 가이드도 참고해 보세요.

네! 티스토리는 WebP를 완전히 지원하고, 네이버 블로그도 WebP 업로드가 가능합니다. 단 플랫폼마다 이미지 용량 제한이 있는데, 압축짤로 미리 줄여두면 업로드 실패 없이 사용할 수 있어요. 자세한 플랫폼별 팁은 플랫폼별 이미지 용량 제한 가이드를 확인하세요.

파일 개수 제한은 없습니다. 다만 한 번에 너무 많은 고해상도 이미지를 올리면 브라우저 메모리 부담이 생길 수 있어요. 일반적으로 20~30장 이하를 권장하며, 대량 작업 시에는 나눠서 처리하는 것이 안전합니다.

네, PNG 포맷을 선택하면 원본 이미지의 투명(알파) 채널이 그대로 유지됩니다. 로고, 아이콘, 스티커처럼 배경이 없는 이미지는 반드시 PNG로 변환하세요. WebP도 투명도를 지원하지만, 호환성이 걱정된다면 PNG가 가장 안전합니다.
